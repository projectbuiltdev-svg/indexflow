TEMPLATE MANAGEMENT ROUTES:

GET /api/we/admin/templates
- Returns all templates including inactive
- { templates[], total }
- Includes usage count per template

POST /api/we/admin/templates
- Creates new template
- Body: { name, category, styleTags, featureTags,
  grapejsState, version }
- Validates grapejsState against block library —
  rejects any block not in we-block-library.ts
- Sets isActive: false on creation
- Logs to we_audit_log

PATCH /api/we/admin/templates/:templateId
- Updates template metadata
- Can update isActive to activate or deactivate
- Incrementing version creates new record —
  existing projects keep old version
- Logs to we_audit_log

DELETE /api/we/admin/templates/:templateId
- Soft delete only — sets deletedAt
- Templates used by existing projects never
  fully removed — kept for rendering
- Logs to we_audit_log

POST /api/we/admin/templates/:templateId/validate
- Re-runs block library validation
- Returns { valid, unknownBlocks[], warnings[] }

WORKSPACE VISIBILITY ROUTES:

GET /api/we/admin/workspaces
- Returns all workspaces with Website Engine data:
  { venueId, name, tier, projectCount, deploymentCount,
  templateUsage, storageUsed, lastActive }

GET /api/we/admin/workspaces/:venueId
- Category 1 data only — always visible:
  Workspace metadata, tier, domain connections,
  template usage, deployment history, form submission
  counts (not content), storage usage, BYOK status

GET /api/we/admin/workspaces/:venueId/audit
- Returns we_audit_log entries for workspace
- Supports ?action= ?severity= ?from= ?to= filters

POST /api/we/admin/workspaces/:venueId/access
- Category 3 access — requires reason in body
- Body: { reason, dataType }
- dataType: 'grapesjs_state' | 'version_history' |
  'analytics_config'
- Logs to we_audit_log with super admin userId,
  venueId, reason, dataType, timestamp
- Sends email to workspace owner automatically —
  cannot be disabled
- Returns requested data

STATS ROUTES:

GET /api/we/admin/stats
- Platform-wide stats:
  { totalProjects, totalDeployments, totalTemplateUses,
  activeWorkspaces, formSubmissionCount,
  deploymentsByType, topTemplates[] }
